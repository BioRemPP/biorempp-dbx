# ToxCSM Database Schema

> **Formal specification of the ToxCSM Database source schema**

The ToxCSM Database provides computational toxicological endpoint predictions for 370 environmental compounds. It is one of the four source databases integrated by the BioRemPP Database Explorer pipeline, joined to the BioRemPP compound data via the `cpd` column.

---

## Overview

The ToxCSM Database uses a **wide-format schema** optimized for multi-endpoint toxicological analysis. The schema consists of a single table with **66 columns** representing compound identifiers, molecular structure (SMILES, ChEBI), and 31 toxicological endpoint predictions expressed both as categorical labels and numeric scores.

!!! warning "Predictive data only"
    All ToxCSM values are **computational QSAR predictions** generated by machine learning models. They are not experimentally measured toxicity values. They support prioritization and exploratory hypothesis generation within the integrated resource; they do not constitute experimental or regulatory toxicological determinations.

## Identifier Columns

### Column 1: `SMILES`

**Data type:** Character (string)  
**Format:** Simplified Molecular Input Line Entry System  
**Example:** `[Ni+2]`, `c1ccc2ccccc2c1`, `C[C@@H](Oc1ccc(Cl)cc1Cl)C(=O)O`

**Purpose:** Machine-readable molecular structure representation for cheminformatics  
**Uniqueness:** Unique â€” one SMILES per compound (370 unique)

---

### Column 2: `cpd`

**Data type:** Character (string)  
**Format:** KEGG Compound identifier  
**Pattern:** `C#####`  
**Example:** `C06790`, `C07535`, `C19609`

**Purpose:** Links to KEGG Compound database and BioRemPP compound data  
**Uniqueness:** Unique â€” 370 unique compound IDs  
**Join key:** Used to join ToxCSM data to BioRemPP compounds during ingest

**Validation rules:**

- âś… Must match regex: `^C\d{5}$`
- âś… Must exist in KEGG Compound database

**Coverage:** 370 / 384 BioRemPP compounds (96.4%) have a ToxCSM entry. 14 compounds have no toxicity prediction data.

---

### Column 3: `ChEBI`

**Data type:** Character (string)  
**Format:** ChEBI identifier  
**Pattern:** `CHEBI:#####`  
**Example:** `CHEBI:29035`, `CHEBI:16716`

**Purpose:** Links to the Chemical Entities of Biological Interest database for cross-referencing  
**Uniqueness:** Unique â€” 370 unique ChEBI IDs  

**Cross-references:**

- ChEBI database: `https://www.ebi.ac.uk/chebi/searchId.do?chebiId=CHEBI:#####`

---

### Column 4: `compoundname`

**Data type:** Character (string)  
**Format:** Free text (chemical nomenclature)  
**Example:** `Naphthalene`, `Benzo[a]pyrene`, `Trichloroethene`

**Purpose:** Human-readable compound name for display and search  
**Uniqueness:** Unique â€” 370 unique names

## Role in Integration Pipeline

The ToxCSM database is joined to BioRemPP data during ingest via the `cpd` column:

```
toxcsm_db.cpd â”€â”€â†’ biorempp_database_v1.1.0.cpd
```

For each compound in BioRemPP, the pipeline:

1. Looks up the matching ToxCSM row
2. Extracts SMILES and ChEBI identifiers into `compound_summary.smiles`
3. Computes `toxicity_risk_mean` as the mean of all non-null value columns
4. Counts `high_risk_endpoint_count` as the number of endpoints labeled "High Toxicity"
5. Stores a JSON toxicity score map in `compound_summary.toxicity_scores`
6. Inserts one `toxicity_endpoint` row per endpoint (31 rows per compound)

## Interpretation Boundaries

!!! warning "Computational predictions require cautious interpretation"
    - **Values are model outputs**, not experimental measurements
    - Higher scores indicate stronger predicted probability of the effect, not confirmed biological activity
    - Predictions should be interpreted in the context of compound class, model training data, and experimental literature
    - ToxCSM predictions support **exploratory risk prioritization** â€” they do not determine real-world toxicity or regulatory compliance

## Loading Examples

=== "Python"

    ```python
    import pandas as pd

    tox = pd.read_csv("data/raw/toxcsm_db.csv", sep=";", dtype=str)
    print(tox.shape)  # (370, 66)

    # Numeric endpoint values
    value_cols = [c for c in tox.columns if c.startswith("value_")]
    tox_num = tox[value_cols].apply(pd.to_numeric, errors="coerce")

    # Compounds with highest mean toxicity score
    tox["mean_value"] = tox_num.mean(axis=1)
    tox.nlargest(10, "mean_value")[["cpd", "compoundname", "mean_value"]]

    # Filter compounds with high hERG prediction
    high_herg = tox[tox["label_hERG_I_Inhibitor"] == "High Toxicity"]
    print(len(high_herg))
    ```
